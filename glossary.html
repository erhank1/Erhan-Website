Glossary

1. CSS – The CSS (Cascading Style Sheets would be a code for the website developers, they would use this code to design and change the way of the website for e.g. set background colour and change the code.

2. Domain – The domain would be a website that whichever the user would like to visit a certain website they can able to type the websites domain for e.g. bbcsports.com and able to access it on the internet browsers.

3. Favicon – This would be a small image that would be related with certain websites for e.g. they would use this to alternative to the text to illustrate to the visitors.

4. Hosting –  This would-be web servers hosting the website for e.g. the website data’s would be housed, served also preserved. It would also allow the website running so that the visitors can able to visit anytime and any devices.

5. HTML – The HTML (Hyper-Text Markup Language) is a website development language, with this developers can able to develop and change the way of the website, this would be mostly used with CSS, JavaScript coding.

6. Meta Tag – This would be HTML tag that would store data’s of the webpage like the owner of the webpage or description of the website. The search engines would use this to show the user that whichever they have entered to able to visit the right website.

7. Navigation bar– The navigation would be used to navigate the visitors from one webpage to another, they can able show with favicons or texts to navigate the visitors.

8. Wireframe – This would show the substance of a webpage deprived of design elements. This would be mostly used before the designing of the website because it would be better for the developers for focusing on the layout of the content and the order without being diverted by the design.

9. CMYK Colour Model: This would be a colour model which would be used for print design this is because it is the way that the paper captivates the brightness and its stands for (Cyan, Magenta, Yellow and Black).

10. Colour Wheel: A colour wheel would show the relationships between different sets of colours for e.g. primary, secondary and tertiary sets of colours.

11. Colour Schemes: This would be a mixture of two or more colours from the colour range.

12. Complementary Colours: The colours that would be adverse each other on the colour wheel for e.g. red to green or blue to orange.

13. Compression: This is a way that the users would free up the space from their computers or images by compressing the file sizes to disgard any extra of data.

14. Contrast: This would highlight the variance of the colours, shapes or other designs.

15. Crop: This would be used to remove any excess part of an image and to be resized to the user’s choice.

16. Dots per Inch (DPI): DPI would be used to clarify the amount of (Dots Per Inch) when printing images. The more DPI the better the resolution.

17. PDF: The PDF would be used for illustrate the documents also the presentations.

18. Pixel: A smallest component of image on the PC’s.

19. Pixels per Inch (PPI): To measure the how much pixels in the resolution of the image.

20. PNG: A image format that has see-through background.

21. Watermark: The watermark would be mostly used on the images this would let the other users know about the owner of the image that they are looking of it, this would be done because of stop stealing images without content.

22. HTML Tags: This would be used in HTML language and it defines the component of the assemble of a webpage.

23. HEX Colours: Would define the colours with hashtag symbol in the beginning and the six numbers fallowing it that would create a specific colour and it is used mostly in developing the webpages.

24. Responsive Web Design: A feature that the website would be changed when the user would access it on many devices such as using computer would show a desktop version of the website and using smartphone would be mobile version of the website which would suit whichever the size of the screen.

25. Page views: This would show the owner of the website how many people has accessed into the website.

26. Direct Search: If a user would know exact URL to website that they would like to access it to then it would take the user straight to that website without any other websites to go through.

27. Domain registration: The domain would be a website that whichever the user would like to visit e.g. bbcsports.com. To able register the website the user needs to able to have an ownership to that domain otherwise they cannot use that domain name for e.g. they cannot use bbc.co.uk because it has already had ownership by the BBC.

28. Index/Homepage: This would be first page of the websites.

29. ISP: The Internet Service Provider would deliver to the location of the user to be connected to the internet.

30. JavaScript: Would be a scripting language would be used to allowing to animate the websites.

31. Updates: The updates would be a maintenance of a specific software or to update changes to the webpage.

32. Search Engine: Would provide a service to the user to help them find a website, image, videos and many more things that the user is looking for.

33. Search Engine Optimisation: This would allow the website to be seen an the top of the search results, the first page of results would be seen much more than others.

34. Title: When a user would visit a webpage, they can able to observe the webpage then they can able to save it with the page title.

35. Blog: A blog would be a web log text where the user can able to publish their text on the internet and that would be consist only by one author.

36. Cache: Cache would be a memory storage component that is aimed to fast retrieval of the frequently used data.

37. Click Through Rate (CTR): The Click Through Rate would be a ratio of number of clicks to the section of impressions. It then would show to the user the result of it for e.g. when advertising your business into the internet you can see how many people that viewed the advert and this would count as impressions.

38. Cloud Storage: A Cloud storage would be a storage in the internet for the users, they can able to store many of types of data such as documents, images, videos and much more. The servers would be looked after by the cloud storage provider and it would be managed and maintained by them.

39. Cookies: Cookies would be a memory of storage that would be used to recover mostly used data for e.g. if a person logs in to a website and next time they want to log in again they can able to see some of their log in information that they would like to use it again if necessary. This would able to speed up the process of trying to log in to a website.

40. Cost Per Click (CPC): When the user would click into an advert the user would be charged for the per click and the advert provider would be earning from it.

41. JPEG: Joint Photographic Experts Group would be defined as the image file type, it is a type of certain degree of image compression at the outlay of the image quality. The JPEG cannot be resized because it can able to loose the image quality very quickly.

42. jQuery: Is a collection of functions using JavaSciprt and it would able to provide many different types of animations for the webpages for e.g. many of the webpage slideshows would be created by this.

43. Dedicated Hosting: This would be form of a website that the hosting of the account owner would have all of the control of the web server and do not divide the server with other users.

44. Domain Name System (DNS): Would be a naming web IP addresses to a name server for e.g. when a user searches a skysports.com in the search engine the DNS would translate this into IP address and links the user straight to the website.

45. E-Commerce: Nowadays there are millions of e-commerce on the World Wide Web, the meaning of e-commerce would be a website that sells products online to the users and allows the user to pay for the product in the webpage.

46. File Transfer Protocol (FTP): Would be a consistent protocol that would used to assign the data’s between the computers it is mainly used to upload and download the data’s from the web servers.

47. Firewall: This would be a security for the computers that would stop any unapproved access to the computers.

48. Flash: Is a animation software tool that would create complex level of games, video players and animations.

49. Graphics Interchange Format (GIF): A image file format for still and dynamic images.

50. Mouse over: Would refer as an event that the user would moving their mouse cursor to the specific component and it would able to do many things such as changing colour, animation and much more.
